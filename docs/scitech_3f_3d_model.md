# 科学技術高校3F 2.5D/3Dモデル

## この文書の範囲

この文書は、科学技術高校3F平面図を2.5D階層データと3D表示へ変換する際の抽出規則、縮尺、階情報、校正根拠をまとめたものです。

基本的な色の意味、サンプルの読み込み手順、推奨する開始位置・出口・火元については [`scitech_3f_sample_map.md`](./scitech_3f_sample_map.md) を参照してください。同文書が単一フロア検証の説明であるのに対し、本書は実装済みの階層・階段・3D座標への変換を説明します。

## 元画像

元画像はユーザー提供の校舎3F平面図です。

| 項目 | 値 |
| --- | --- |
| 元画像寸法 | 600 × 800 px |
| 色形式 | 24-bit RGB PNG |
| サンプル格納先 | `sim/assets/maps/scitech_3f_walkable.png.base64` |
| ローダー | `sim/js/sample3f-loader.js` |
| 抽出profile | `SCITECH_3F_PROFILE` |

色は次の意味で扱います。

- 白: 廊下・通路
- 黒: 部屋、壁、通行不可領域
- 緑: 屋内階段または外階段
- 黄: 出口（黄色の連結領域ごとに1地点へ集約）
- 白い文字: 室名・階段名などの注釈であり、通路ではない

元画像は2D画面の背景としても使えますが、経路探索には画像をそのまま使わず、次節の手順でwalkable/stair templateへ変換します。

## 600×800 pxから150×200セルへの変換

1セルは画像上の4×4 pxです。このためグリッド寸法は次になります。

```text
gridWidth  = floor(600 / 4) = 150
gridHeight = floor(800 / 4) = 200
```

`extractScitech3FGridFromImageData()` は既存2Dと同じく、各4×4 pxブロックの中心画素を既定で使います。これにより既存グリッドとの座標互換を保ちます。細線図向けには `sampleMode: "coverage"` でブロック内占有率判定へ切り替えることもできます。

判定の概略は次のとおりです。

1. 黄成分が十分強い画素を出口候補とする。
2. 緑成分が赤・青より十分強い画素を階段候補とする。
3. RGB各値が白閾値（既定200）を超える画素を通路候補とする。
4. 中心画素が黄なら出口セル、緑なら階段セル、白・緑・黄なら通行可能セルにする。
5. 白・緑・黄で構成される4近傍連結成分を列挙する。
6. 最大成分、緑または黄を含む成分、十分大きく画像端の細線ではない白成分を残す。

最後の処理により、廊下網から離れた室名文字や数字は小さな連結成分として除去されます。一方、1Fのように複数の正規領域があるマップは、緑階段または黄色出口を含む各領域を保持します。緑階段は通路と同じ連結判定へ入れるため、白だけを抽出した場合のように階段部分で経路が切れません。

出力は主に次の4つです。

- `walkableTemplate[cy][cx]`: 通路、階段または出口なら `true`
- `stairTemplate[cy][cx]`: 緑から抽出した階段なら `true`
- `exitTemplate[cy][cx]`: 黄から抽出した出口領域なら `true`
- `exitPoints`: 黄色の4近傍連結領域ごとに選んだ代表セル

黒領域と除去された文字領域は `walkable = false`、`wall = true` の基礎になります。抽出処理は部屋名を認識したり、黒い部屋内部を自動的に床へ変換したりはしません。

## 抽出される5つの階段領域

4pxグリッド上では、緑は5つの独立した矩形領域として現れ、白い廊下を含む最大連結成分へすべて接続しています。中心画素による校正調査で得た概数は次のとおりです。占有率判定では境界セルが1セル程度増減する場合があります。

| 位置 | おおよそのグリッド範囲 | 概数 |
| --- | --- | ---: |
| 上側左階段 | `x=20..28, y=66..79` | 126セル |
| 上側右階段 | `x=118..125, y=65..79` | 120セル |
| 中央右階段 | `x=91..102, y=90..104` | 180セル |
| 下側左階段 | `x=39..51, y=142..151` | 130セル |
| 下側右の外階段 | `x=126..131, y=157..170` | 84セル |

緑セルは自動的に `stair = true`、`walkable = true` になります。右下領域は外階段として扱い、他は既定で屋内階段とします。

階段領域の抽出と階段リンクは別の概念です。3F画像から階段セルを抽出しても、接続先階のセルは画像だけでは決められません。2Fまたは4F側に対応する階段セルを用意し、`from` と `to` を明示的にリンクしてください。

## 廊下幅によるcellSize校正

提供された実寸情報では、上部廊下の全幅が約2.9m、中心から片側の壁までが約1.45mです。

4pxグリッドを調べると、上部廊下の典型的な厚さは7セルです。上部廊下の直線部は `y=60..66` を占め、交差部や階段接続部を除く大多数の列で7セル厚になっています。

したがって廊下幅だけから求めた値は次です。

```text
cellSizeMeters = 2.9 m / 7 cells
               = 0.414285... m/cell
```

設定値として扱いやすいよう、科学技術高校3F profileの既定値は次とします。

```text
cellSizeMeters = 0.42 m
```

この値では7セルの廊下は2.94m、半幅3.5セルは約1.47mとなり、2.9m / 1.45mという実寸とよく一致します。既存シミュレーター全体の一般的なfallback値0.5m/cellは残し、3Fサンプルを読み込んだときだけprofile値0.42mを設定します。

## 縦方向65.6mとの不一致

提供されたもう1つの基準は、縦方向の代表長さ約65.6mです。一方、抽出した通路成分の縦spanは次のとおりです。

| 測定対象 | グリッド範囲 | セル数 | 65.6mから逆算したcellSize |
| --- | --- | ---: | ---: |
| 外階段を含む最大成分全体 | `y=60..170` | 111 | 約0.591m |
| 建物内の中央縦通路 | `y=60..157` | 98 | 約0.669m |

セル中心間の距離として数える場合はそれぞれ110区間、97区間で、約0.596m、約0.676mになります。いずれも廊下幅から求めた約0.414m/cellとは一致しません。

これは元画像が正確な等縮尺CAD図ではなく、室名、階段、廊下を説明する模式図だからです。しかも廊下幅と縦長さは画像上の同じY軸を使うため、X/Y別の単純な異方性スケールでも両方を同時には満たせません。

この実装では、避難者と壁の見た目・通行幅へ直接効く廊下幅を優先して0.42m/cellを既定とします。65.6mは全体スケールの校正候補として保持し、`cellSizeMeters` をUIまたはprofileで変更可能にします。

選択の目安は次のとおりです。

| 目的 | cellSizeMeters | 7セル廊下 | 最大成分111セル |
| --- | ---: | ---: | ---: |
| 廊下幅2.9mを優先 | 0.42 | 2.94m | 46.62m |
| 従来fallbackとの互換 | 0.50 | 3.50m | 55.50m |
| 全spanを65.6mに合わせる | 0.591 | 4.14m | 約65.6m |
| 建物内98セルを65.6mに合わせる | 0.669 | 4.68m | 74.26m |

精密な距離・所要時間校正が必要な場合は、元CAD寸法に基づいて通路中心線グラフを別途校正するか、通路幅を保ったまま画像の縦方向配置を再作図してください。

## 3Fの階情報と3D座標

floor indexは0始まりです。したがって3Fは `floorIndex = 2` です。既定階高3.5mを使うと床標高は7.0mになります。

```js
{
  floorIndex: 2,
  name: "3F",
  zMeters: 7.0,
  elevationMeters: 7.0,
  floorHeightMeters: 3.5,
  wallHeightMeters: 2.8,
  gridWidth: 150,
  gridHeight: 200,
  cellSizeMeters: 0.42
}
```

3D座標への変換は次です。

```text
worldX = cx * 0.42
worldY = 7.0
worldZ = cy * 0.42
```

`zMeters` は既存floorデータとの互換フィールドで、3D renderer上の鉛直座標は `worldY` です。平面図のY方向は3D空間の `worldZ` へ写します。

壁は非通行セルと通路境界から高さ2.8mの立体として表示します。階高3.5mとの差0.7mは床スラブ、天井裏、表示上の階間余白を含む扱いです。これは詳細な構造モデルではなく、階の識別と煙・人の表示を分離するための値です。

0.42m/cellの場合、グリッド全体の論理寸法は約63m × 84mです。ただし黒い余白を含む画像全体の寸法であり、実際に歩行可能な最大成分のbboxはおよそ140 × 111セルです。

## 壁・床・階段の表示規則

- 通行可能セル: 明るい床として表示する。
- 非通行セル: 床を作らない、または通路境界に暗い壁を立てる。
- 壁高: 既定2.8m。
- 階段セル: 緑系または階段用形状で表示する。
- 外階段: 屋内階段と区別し、煙の換気係数も変える。
- 火災セル: 床面上の炎として表示し、通行不可にする。
- 煙: `smokeDensity` とvisibilityに応じた半透明表現とする。

元画像の室名文字は床・壁形状の決定には使いません。部屋名や設備名を3Dラベルとして表示したい場合は、画像認識ではなく別のannotationデータとして追加するのが安全です。

## 複数階への拡張

現在のサンプル画像は3Fだけですが、データ構造は複数階に対応します。

1. 追加階も同じグリッド寸法へ正規化する。
2. 各floorへ固有の `floorIndex`、`elevationMeters`、walkable templateを設定する。
3. 3Fの5階段領域に対応する位置を追加階でも階段セルにする。
4. 実際に接続する組だけを階段リンクで結ぶ。
5. 屋外・屋内・非常階段の型、移動時間、容量、煙移送係数を設定する。

単純に3Fグリッドを他階へ複製することは構造テストには利用できますが、実校舎の避難評価では各階の平面図、出口、教室配置を個別に設定してください。

## FDS CSVとの座標合わせ

FDS値をこのモデルへ重ねる場合、各recordを最低限次のキーへ対応付けます。

```text
(timeSec, floorIndex, cx, cy)
```

元FDS座標がメートル単位の場合は、floor原点と `cellSizeMeters` を使って `cx/cy` へ変換します。0.42mと縦65.6m基準では座標が大きく変わるため、FDS連携前に採用した校正値を固定し、CSV側の原点・向き・単位を記録してください。

FDSに値がないセル・時刻は軽量fallbackを使います。FDS値がある場合は heat flux、temperature、optical density、CO、visibilityなど、存在する物理量だけを優先します。

## 検証チェックリスト

- 読み込んだ画像が600×800 pxである。
- 抽出グリッドが150×200セルである。
- 白い通路と緑階段が同じ最大連結成分へ入る。
- 室名文字が独立歩行セルとして残っていない。
- 緑領域が5か所として認識される。
- 上部廊下が典型7セル幅である。
- サンプル読込後の `cellSizeMeters` が0.42mである。
- 3Fが `floorIndex = 2`、標高7.0mで表示される。
- 壁高が2.8m、階高が3.5mで表示される。
- 右下外階段が屋内階段と区別される。
- 階段セルだけでなく、必要な上下階リンクが設定されている。
- 2Dと3Dで同じ出口、火災、煙、エージェント位置が見える。
